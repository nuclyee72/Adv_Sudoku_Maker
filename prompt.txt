변형 스도쿠가 플레이 가능한 html 만드는 계획을 세워보자

오브젝트:
흰색 배경, 왼쪽-가운데 부분 배경 위 띄워져 있는 스토쿠 게임 판(이동 가능), 오른쪽 배경 위 띄워져 있는(이동 가능) 숫자 입력 키패드

조작:
스도쿠 게임 판 이동 - WASD
스도쿠 게임 판 확대 - 마우스 휠 위(확대) 아래(축소)
칸 선택 - 마우스 좌클릭(default = 가장 왼쪽 위 칸)
칸 이동 - 키보드 방향 키

스도쿠 판의 구조:
유연한 게임 변형을 위해 표 구조로 접근 (2x2 배열)
기본 구조들: 3x3구조체(작은 박스, 1-9까지 들어가야 함, 작은 박스의 왼쪽 위 칸을 기준), 9x9구조체(큰 박스, 각 가로, 세로에 중복되지 않아야 함, 큰 박스의 왼쪽 위 칸을 기준)
추가 룰을 가진 구조들: 나중에 추가 예정
칸의 형태는 정사각형이나 직사각형으로 고정되지 않음, 할당된 칸이 없는 좌표는 투명하게 두어 배경이 보이게 함

룰:
각 구조의 룰에 어긋난 숫자들이 존재하는 경우 그 숫자들이 빨간색으로 바뀜, 해결 시 다시 원래 색으로 돌아옴

계획 세워보자

스도쿠 보드 패널은 패널로 있지 말고 스도쿠 판만 떠 있게 해줘, 초기화 버튼의 텍스트 지우기로 바꾸고 숫자 입력 칸으로 옮겨줘

숫자 입력 칸 드래그로 움직이지 않아 고쳐줘
숫자 일벽 칸의 123456789+지우기 키패드 만들어줘 
예시로 9x9 기본 스도쿠 판 으로 세팅해줘
화면 하단 도움말 없애줘

1. wasd로 이동할 때 뚝뚝 끊겨 부드럽게 움직이게 해줘
2. 마우스 드래그로 움직일 때 게임 판을 잡아도 이동되게 해줘
3. 마우스 휠로 확대할 때 마우스 커서를 기준으로 확대되게 해줘

게임 판보다 숫자 입력 창이 더 위에 있게 해줘
숫자 입력 창의 '숫자 입력' 텍스트 없애줘

마우스 휠 확대 아직 조금 두둑두둑 하면서 움직이는데 해결법 있어?

방향키로 움직이는거 속도 조금 늘려줘
그럼 테스트로 4개의 게임판이 각 모서리를 공유하여 십자 모양인 형태의 테스트 게임판 생성해줘 + 답안 하나 만든 뒤 절반정도 비워줘

1. 현재 게임 판이 움직일 수 있는 범위가 제한되어 있음(화면 밖으로 나오면 안됨) -> 최소 한 칸 만 보이는 정도까지 화면 밖으로 나갈 수 있게
2. 전체 범위? 사각형? 게임 판이 움직일 수 있는 범위를 화면 전체로 해줘 현재 화면 전체보다 적음
