# 변형 스도쿠 플레이어 HTML 구현 계획

## 개요

순수 HTML + CSS + JavaScript로 구현하는 변형 스도쿠 게임입니다.
향후 다양한 변형 룰 추가를 고려해 **유연한 구조** 설계를 목표로 합니다.

## Open Questions

> [!IMPORTANT]
> 아래 항목들은 설계에 영향을 주므로 확인이 필요합니다.

1. **첫 번째 변형 스도쿠의 종류**: 일반 9×9 스도쿠를 기본으로 먼저 구현할까요? 아니면 특정 변형(예: 사무라이, 킬러, X-스도쿠 등)을 바로 만들까요?
2. **퍼즐 생성 방식**: 직접 퍼즐을 설계해서 넣을 계획인가요, 아니면 자동 생성(알고리즘) 방식을 원하시나요?
3. **칸의 형태**: 정사각형·직사각형이 아닌 칸의 예시가 있으신가요? (예: 헥사곤, L자형 영역 등) — 이에 따라 좌표 시스템이 달라집니다.
4. **숫자 범위**: 기본 1–9만 사용하나요, 아니면 변형에 따라 다른 범위가 필요한가요?
5. **저장/불러오기** 기능이 필요한가요?

## 아키텍처 설계

### 핵심 데이터 모델

```
GameBoard
├── cells: Cell[][]         // 2D 배열 (좌표 기반)
├── structures: Structure[] // 적용된 구조체 목록
└── rules: Rule[]           // 적용된 룰 목록

Cell
├── coord: {row, col}
├── value: number | null
├── isGiven: boolean        // 초기 주어진 숫자인지
├── isVisible: boolean      // 투명 칸 여부
└── candidates: Set<number> // 후보 숫자

Structure (인터페이스)
├── type: 'box3x3' | 'row9' | 'col9' | 커스텀
├── cells: coord[]          // 포함된 칸 좌표
└── validate(): CellConflict[]

Rule (인터페이스)
└── check(board, cell): boolean
```

### 구조체 시스템 (확장 가능)

| 구조체 | 설명 | 룰 |
|--------|------|-----|
| `Box3x3` | 3×3 작은 박스 | 1–9 중복 없음 |
| `Row9` | 9×9의 각 가로줄 | 1–9 중복 없음 |
| `Col9` | 9×9의 각 세로줄 | 1–9 중복 없음 |
| `CustomRegion` | *(향후)* 불규칙 영역 | 커스텀 룰 |

## 주요 구현 세부 사항

### 1. 게임판 렌더링 방식

**두 가지 후보:**

| 방식 | 장점 | 단점 |
|------|------|------|
| **HTML Table / Grid** | 단순, CSS 스타일 쉬움 | 비직사각형 칸 처리 어려움 |
| **SVG** | 모든 형태의 칸 표현 가능, 확대·축소 자연스러움 | 초기 구현 복잡도 높음 |

> [!TIP]
> 향후 비직사각형 칸을 지원해야 하므로 **SVG 방식**을 권장합니다.
> 각 칸은 `<rect>` 또는 `<polygon>`으로 표현하고, 텍스트는 `<text>`로 오버레이합니다.

### 2. 이동 가능한 패널 (DragPanel)

- 두 패널(게임판, 키패드) 모두 **마우스 드래그**로 자유롭게 이동
- 게임판은 추가로 **WASD 키**로 이동
- 게임판은 **마우스 휠**로 확대/축소 (SVG viewBox 조작)
- `z-index` 기반 포커스 관리 (클릭한 패널이 앞으로)

### 3. 칸 선택 및 내비게이션

- 마우스 클릭으로 칸 선택 (기본값: 최상단 좌측 가시 칸)
- 키보드 방향키로 인접 가시 칸으로 이동
- 선택된 칸 하이라이트 + 같은 숫자 칸 연하게 하이라이트

### 4. 충돌 감지 (Validator)

```
입력 발생
  → Board.setValue(coord, value)
  → Validator.check(board, coord)
    → 각 Structure.validate() 순회
    → 충돌 좌표 Set 반환
  → BoardRenderer.updateConflicts(conflictSet)
    → 충돌 칸의 숫자 색상을 빨간색으로 변경
    → 충돌 해소 시 원래 색상 복원
```

### 5. 숫자 입력 키패드

- 1–9 버튼 + 지우기(X) 버튼
- 마우스 드래그로 이동 가능
- 키보드 숫자키(1–9), Delete/Backspace로도 입력 가능

## UI / 디자인

- **배경**: 흰색 (`#FFFFFF`)
- **패널**: 유리형태(glassmorphism) 부유 패널, 미세한 그림자
- **색상 팔레트**:
  - 선택 칸: 파란색 계열 하이라이트
  - 같은 숫자 칸: 연한 파란색
  - 충돌 숫자: `#E53E3E` (빨간색)
  - 초기 제공 숫자: 진한 회색, 볼드
  - 입력 숫자: 파란색 계열
- **폰트**: Google Fonts - `Inter` 또는 `Outfit`
- **애니메이션**: 충돌 발생 시 살짝 흔들리는 효과, 패널 이동 시 부드러운 트랜지션
